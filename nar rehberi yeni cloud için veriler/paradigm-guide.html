Imperative / Procedural
Do this, then this, then this.
You naturally think in sequences — and that's exactly what procedural programming is. You write instructions line by line, like a recipe. The computer follows them in order, just like you'd explain something step-by-step to a friend.

Imagine making a cup of tea. You'd think: boil water → add tea bag → wait → remove bag → drink. Procedural code looks like that: # Make a cup of tea boil_water() add_tea_bag() wait(3) # minutes remove_tea_bag() print("Tea is ready!") Simple, linear, readable. This is the most natural place to start.
👍 Grows With You
Object-Oriented (OOP)
Think in things that have properties and behaviors.
Once you're comfortable with steps, OOP lets you bundle related data and actions together into "objects." Think of a Dog object that knows its name and can bark. It's a great next step once procedural feels natural.

▶ See a real example (click to expand)
A Dog object bundles together what it is (name, breed) and what it can do (bark, sit): class Dog: def __init__(self, name): self.name = name def bark(self): print(f"{self.name} says: Woof!") my_dog = Dog("Biscuit") my_dog.bark() # Biscuit says: Woof! Start with procedural first — OOP will feel much more natural after a few months.
🔭 Explore Later
Functional Programming
Describe transformations, not steps.
Functional programming is powerful but abstract — it's less about how to do something and more about what should happen. Great once you have experience, but it can feel counterintuitive at first.
